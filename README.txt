ШТАБ — готовый проект с backend

Что внутри
- Фронт:
  index.html
  register.html
  login.html
  onboarding.html
  organizations.html
  app.html
  settings.html
  assets/styles.css
  assets/app.js
- Backend:
  backend/app/main.py
  backend/requirements.txt
  backend/start_backend.bat
- Скрипт запуска фронта:
  start_frontend.bat

Что уже работает
- регистрация
- вход
- logout
- создание организации
- загрузка CSV
- демонстрационные данные
- список организаций
- рабочая панель
- настройки компании
- приглашения пользователей
- состав команды
- история импортов на backend в sqlite
- хранение данных в sqlite на backend
- базовый invite flow с токеном приглашения и endpoint принятия
- базовые ограничения по ролям для чувствительных действий

Как запускать
1. Backend:
   открой backend\start_backend.bat
   или вручную:
   cd backend
   py -m venv .venv
   .venv\Scripts\python.exe -m pip install -r requirements.txt
   .venv\Scripts\python.exe -m uvicorn app.main:app --reload

2. Frontend:
   открой start_frontend.bat
   или вручную в корне проекта:
   py -m http.server 8080

3. Открой в браузере:
   http://127.0.0.1:8080

Важно
- Эта версия сделана максимально стабильной для локального использования на Windows.
- Для production потом понадобятся HTTPS, почта, полноценный invite UI, деплой и усиление безопасности.

Что исправлено в этой версии
- Исправлено падение assets/app.js на страницах, где нет кнопки applyCsvReplace.
- История импортов больше не хранится только в браузере: она пишется в таблицу product_imports в SQLite.
- Добавлен GET /api/v1/import-history?organization_id=... для загрузки истории импортов с backend.
- В настройках для тарифа Старт team-блоки скрываются до любых team API-запросов.
- Смена тарифа убрана из обычного обновления организации: тариф меняется только через billing flow.
- В список организаций backend теперь отдаёт features по тарифу, чтобы фронт не рассинхронизировался с лимитами.
- Добавлен logout endpoint и срок жизни сессии; истёкшие сессии очищаются автоматически.
- Добавлены индексы и уникальные ограничения для memberships, sessions, invites и product_imports.
- Приглашения теперь получают token, created_at, accepted_at и могут быть приняты через POST /api/v1/invites/accept.
- Чувствительные действия доступны только роли Руководитель: изменение организации, приглашения, импорт CSV, биллинг.
- Усилена валидация CSV: проверка дублей SKU/id, пустых channel/warehouse, отрицательных значений и диапазона rate-полей.
- В настройках добавлен предпросмотр первых 5 строк CSV перед импортом.
- После импорта settings.html показывает факт обновления backend-истории.
- Параметр paid_until теперь заполняется после подтверждения оплаты.

Ограничения текущей версии
- Импорт товаров пока работает в режиме replace. Backend уже сохраняет mode импорта, но upsert ещё не включён.
- Invite flow пока API-only: токен возвращается в ответе API, а отдельной публичной страницы принятия приглашения ещё нет.


Что добавлено в этой версии
- Расширена юнит-экономика: маржа %, ROAS, ACOS, цена безубыточности, рекомендованная цена, прогноз прибыли на 30 дней.
- На панели добавлены прогноз на 30 дней и центр уведомлений по закупкам, марже и рекламе.
- В импорт CSV добавлено сравнение с текущими данными до применения и статистика изменений после импорта.
- Добавлен журнал действий (audit log) по организации, оплате, приглашениям и импорту товаров.
- Добавлена публичная страница invite.html для принятия приглашения по ссылке.
- CORS ограничен локальными origin вместо режима allow_origins=["*"].
